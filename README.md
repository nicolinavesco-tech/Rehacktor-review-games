Rehacktor –  Game Reviews
Project Overview

This project represents an important step in my journey as a frontend developer.

With Rehacktor, I moved beyond building simple interfaces and started working on a more complete application, where different parts of development come together: API integration, authentication, state management, and interaction with a real database.

Throughout this project, I focused on improving how I structure a React application, how I manage data flow, and how I think about user interaction in a more practical and realistic way.

While building this project, I had the chance to work on:

Fetching and managing data from an external API
Creating a dynamic and responsive UI
Structuring a scalable React project
Implementing client-side routing
Handling forms and user input
Managing authentication and user sessions
Connecting a frontend app to a real backend service

I also improved my ability to break down features into reusable components and organize code in a cleaner and more maintainable way.

This project helped me understand how a real application works under the hood.

In particular, I improved my skills in:

Thinking in terms of application architecture, not just components
Managing global state and user data
Handling asynchronous operations and API calls
Working with authentication flows
Interacting with a database and understanding data relationships
Debugging and solving real-world issues

It was also my first time working with features like user profiles, favorites, and reviews, which made the project feel much closer to a real product.

To build this project, I used:

React (Vite) for building the frontend
React Router for navigation and routing
Tailwind CSS + DaisyUI for styling and layout
Supabase for authentication, database, and storage
RAWG API for retrieving video game data
React Hook Form for handling forms
React Icons for UI elements

Each of these tools helped me understand different aspects of modern web development and how they work together in a real project.

The project is still a work in progress.

I’m continuing to refine both the code and the user experience, and I plan to expand it with additional features and improvements.

This project is not just something I built once — it’s something I’m actively using to grow and experiment as a developer.

Rehacktor is more than just a project — it’s a reflection of my progress.

It shows how I’ve improved in structuring applications, working with real data, and thinking more like a developer rather than just writing code.
